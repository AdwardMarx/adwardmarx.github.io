5.GitHub Pages仓库没有.md文件
当运行了hexo generate并执行 hexo deploy的时候，Hexo 会：将Markdown源文件（.md）转换为静态HTML页面。
把这些HTML文件推送到你设定的GitHub Pages仓库。推送到GitHub上的是生成后的站点内容（public/ 文件夹）。原始的 .md 文章其实是保存在你本地的 source/_posts/ 目录下，但它们没有被推送到 GitHub 上。
在我更换电脑后，发现当时没有把博客的.md推送到GitHub上，所以换电脑就找不到原来的文件了。为了防止这种情况，一般采用两个GitHub仓库。
部署仓库：只保存public/生成的HTML内容，供GitHub Pages展示。
源码仓库：保存 Hexo 的全部源代码，包括 .md 文件、主题、配置等。
